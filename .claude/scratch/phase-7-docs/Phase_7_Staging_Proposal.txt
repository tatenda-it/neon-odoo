## Why staging
Robin's Phase 7 Open Questions answers (locked 20 May 2026) chose the maximalist option across most groups: org-wide user coverage (A2=B), all four certification categories including soft skills (B1=D), formal approval workflow for external training (E2=A), Knowledge Base as core scope (F1=A), and layered three-tier event_job assignment gating (G2=D). The result: estimated 19–24 milestones across the full phase — roughly twice the scope of Phase 6.
Building this as a single linear phase means Robin sees no production value until ~8 weeks of build complete. Sub-phase staging means each major capability ships independently as it's ready. Phase 7a (the foundation) delivers operational value in ~3 weeks instead of 8.
This proposal breaks Phase 7 into four logical sub-phases. Each ships as a tagged production release, gets walkthrough acceptance, then the next sub-phase begins design. Master plan numbering stays at 12 phases — sub-phases are tracked internally.

## Order of build
Suggested execution order: 7a → 7b → 7c → 7d. Reasoning:
7a anchors the certification model that 7b's onboarding, 7c's external bookings, and 7d's permissions all depend on.
7b lands second so crew portal exists for crew to see their own records as they're recorded by 7c bookings.
7c lands third — by then the crew records and certification model are settled enough for external bookings to feed cleanly into them.
7d lands last because (a) it's the largest individual sub-phase and (b) it's the most independent — KB content doesn't depend on certifications being in place, just on the institutional infrastructure being settled.

Each sub-phase ends with a tagged production release and a Robin/Munashe walkthrough — same shape as Phase 6's M12. Decisions about polish or refinement land before the next sub-phase opens.
