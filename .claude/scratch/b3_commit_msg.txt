feat(b3): AI deployment plan generation

P-B3 -- AI-generated deployment plan with strict design-seed lock.
Routes document generation through B13's ClaudeDocGenAdapter
(neon_doc_gen) for narrative + structure + sequencing; every fact
(quantities, dates, crew names, conflict deficits) comes from
Python via DeploymentPlanFactGatherer. A pure-Python
DeploymentPlanValidator rejects any LLM output that contradicts or
omits a fact, with seven enforcement rules.

Models:
  neon.deployment.plan                   header + state machine
                                          (draft -> generated ->
                                           reviewed -> final;
                                           regenerate -> superseded)
                                          + revision counter
                                          + source_conflict snapshot
                                          + audit (gen/review/final
                                           timestamps + actors).
                                          perm_unlink=0.
  neon.deployment.plan.call.time.config  singleton (per gate-1 (c))
                                          for the Lisa-tunable
                                          call-time offsets
                                          (chief -30, lead -60,
                                          rest -15 minutes from
                                          dispatch/prep anchor) +
                                          is_ops_signed_off flag.

Python helpers (NOT models -- importable by B4/B5):
  DeploymentPlanFactGatherer  walks event_job -> venue + windows +
                              crew + equipment + B2 conflict snapshot
                              + Python-pre-computed crew call times.
  DeploymentPlanValidator     7 strict rules:
                                R1 hallucinated quantity
                                R2 hallucinated competing event
                                R3 hallucinated crew name
                                R4 OMITTED deficit (cardinal sin)
                                R5 wrong section.key enum
                                R6 split per gate-1 (a): concrete
                                   parseable datetimes HARD-reject
                                   if not in facts; relative phrasing
                                   ("morning of the event") passes
                                R7 data_quality_note verbatim
                                   carry-through
  DeploymentPlanGenerator     orchestrator: gather -> B13 ->
                              validate -> persist; ONE auto-retry
                              with corrective context; on second
                              failure -> quarantine_json + UserError.

Per gate-1 D6: refuses to generate when event_job.state == 'draft'
(mirrors B2-DM-2; demand isn't authoritative until out of draft).

Per gate-1 D8: maps all six B13 typed DocGen* exceptions to
friendly UI errors; no silent fallback.

Per gate-1 D10 (trim): on-screen plan_summary_html render only.
PDF deferred as a fast follow.

Dependency note: neon_doc_gen NOT a hard depends to avoid the
neon_jobs -> neon_doc_gen -> neon_core -> neon_jobs cycle (neon_core
depends on neon_jobs for the cascade pattern). The B13 adapter is
imported LAZILY inside _call_claude with ImportError -> UserError.
